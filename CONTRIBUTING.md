# Contributing to NomadRide 🏍️

First off, thank you for considering contributing to NomadRide! It's people like you who make it a great tool for the biking community.

## 🌈 Our Community Strategy

NomadRide is built with a **cross-platform first** mindset. 

- **Android Primary**: The core maintainers primarily use Linux/Android for development and testing.
- **iOS Welcome**: We highly encourage iOS users and developers to contribute! Even if you don't see frequent iOS-specific updates, the project is built using **React Native (Expo)** to ensure maximum compatibility.

## 🛠️ How to Contribute

### 1. Reporting Bugs
- Check the [Issues](https://github.com/your-repo/nomadride/issues) to see if the bug has already been reported.
- If not, open a new issue with a clear title and description. Include steps to reproduce and your environment details (OS, Node version, Device).

### 2. Suggesting Enhancements
- Open an issue to discuss your idea before implementing it. This ensures it aligns with the project's vision.

### 3. Pull Requests
1. **Fork** the repository.
2. **Create a branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
3. **Commit** your changes following the [Conventional Commits](https://www.conventionalcommits.org/) specification.
4. **Push** to your fork.
5. **Open a Pull Request** against the `main` branch.

## 🏗️ Development Environment

- **Node.js**: v24 or later.
- **Expo**: We use the Expo managed workflow.
- **Terraform**: Required for infrastructure changes.
